# Project Inputs

Source materials, requirements, and input data for project work.

## Contents

- User-provided requirements
- Source documents
- Input datasets
- Reference materials
- Configuration files
- Specifications

## Organization

Organize inputs by:
- Type (requirements, data, references)
- Source (user, system, external)
- Phase or feature area
- Date received

## Management

- Keep originals unmodified
- Reference in session logs
- Version for tracking changes
- Maintain clear lineage to outputs

## Naming Convention

Suggested: `{source}-{type}-{date}.{ext}`

Example: `user-requirements-2026-03-10.md`

## Processing

- Reference inputs in decisions and session logs
- Document any transformations
- Link outputs back to inputs
- Archive after project phase completion
