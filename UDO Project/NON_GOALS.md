# Non-Goals

Explicitly list what this project is NOT trying to achieve.

## Purpose

Defining non-goals is as important as defining goals. This document prevents:
- Scope creep
- Misaligned expectations
- Wasted effort on out-of-scope work
- Unrealistic commitments

## Format

```
## Category: [Non-Goal Name]

**Statement**: Clear, specific statement of what we will NOT do

**Rationale**: Why this is explicitly out of scope

**Boundary**: Where the boundary is (what IS in scope nearby)

**When**: If this could change in future, state conditions
```

## Template

Add your project's non-goals below:

### Example Structure

```
## Process: Automation of [Task]

**Statement**: This project will not automate [specific process]

**Rationale**: [Explain why]

**Boundary**: Instead, [what we will do]

**When**: This may change if [future conditions]
```

## Usage

- Review non-goals at project start
- Reference in decisions when boundaries are questioned
- Update as scope clarifies
- Communicate to stakeholders
